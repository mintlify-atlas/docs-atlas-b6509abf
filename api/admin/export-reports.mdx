## GET /api/admin/reportes/:tipo

Generates comprehensive reports for a specified time period, including user counts, failure statistics, and breakdowns by role-specific attributes.

### Path Parameters

<ParamField path="tipo" type="string" required>
  Report period type
  
  Valid values:
  - `"diario"` - Daily report
  - `"semanal"` - Weekly report
  - `"mensual"` - Monthly report
  - `"semestral"` - Semester report
</ParamField>

### Query Parameters

<ParamField query="offset" type="integer" default="0">
  Time offset for historical reports
  
  - `0` = current period (default)
  - `1` = previous period
  - `2` = two periods ago
  - etc.
  
  **Note:** For `"semestral"` reports, offset is ignored and always returns the active semester.
</ParamField>

### Examples

```bash
# Today's report
GET /api/admin/reportes/diario

# Yesterday's report
GET /api/admin/reportes/diario?offset=1

# This week's report
GET /api/admin/reportes/semanal

# Last week's report
GET /api/admin/reportes/semanal?offset=1

# Current month
GET /api/admin/reportes/mensual

# Active semester
GET /api/admin/reportes/semestral
```

### Period Calculation Logic

**Source:** `src/modules/admin/infrastructure/repositories/AdminRepositoryImpl.js:273-313`

#### Daily (`"diario"`)
- **Current (offset=0)**: Today from 00:00:00 to now
- **Previous (offset=1)**: Yesterday from 00:00:00 to 23:59:59
- **offset=N**: N days ago, full day

#### Weekly (`"semanal"`)
- **Current (offset=0)**: Monday of this week to now
- **Previous (offset=1)**: Previous Monday to Sunday
- **offset=N**: N weeks ago

#### Monthly (`"mensual"`)
- **Current (offset=0)**: First day of this month to now
- **Previous (offset=1)**: Full previous month
- **offset=N**: N months ago

#### Semester (`"semestral"`)
- Always returns data for the active semester (from `semestres` table)
- If no active semester exists, defaults to current half-year (Jan-Jun or Jul-Dec)
- **offset parameter is ignored**

### Implementation Details

**Source:** `src/modules/admin/infrastructure/repositories/AdminRepositoryImpl.js:272-488`

The report generation process:

1. **Calculate date range** based on period type and offset
2. **Execute parallel queries** for:
   - Role counts
   - Failures in period
   - Student programs
   - Employee departments
   - Contractor companies
3. **Cross-reference failures** with role-specific info using ID lookups
4. **Aggregate data** by categories (program, department, company)
5. **Format date labels** for display in Spanish locale
