## Overview

This guide will help you get UCC Control de Acceso running on your local machine in just a few steps. The system consists of a React frontend connected to a Supabase PostgreSQL database.

<Note>
  This quick start guide assumes you have basic knowledge of Node.js, React, and PostgreSQL.
</Note>

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v16.x or higher)
- **npm** (v8.x or higher) or **yarn**
- **Git**
- A **Supabase** account (free tier is sufficient)

## Step 1: Clone the Repository

<Steps>
  <Step title="Clone and Navigate">
    Clone the repository and navigate to the project directory:

    ```bash
    git clone https://github.com/xxxpedrozaxxx/ucc-control-acceso.git
    cd ucc-control-acceso
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>
</Steps>

## Step 2: Set Up Supabase Database

<Steps>
  <Step title="Create Supabase Project">
    1. Go to [supabase.com](https://supabase.com) and sign up or log in
    2. Click **New Project**
    3. Enter project details:
       - **Name**: `ucc-control-acceso`
       - **Database Password**: Choose a strong password
       - **Region**: Select closest to your location
    4. Click **Create new project** and wait for initialization
  </Step>

  <Step title="Run Database Migrations">
    Navigate to the SQL Editor in your Supabase dashboard and execute each file in order:

    1. **01_usuarios.sql** - Creates `usuarios` table
    2. **02_roles.sql** - Creates `roles` and `usuario_roles` tables
    3. **03_info_estudiante.sql** - Creates student information table
    4. **04_info_empleado.sql** - Creates employee information table
    5. **05_info_contratista.sql** - Creates contractor information table
    6. **06_triggers_roles.sql** - Creates automatic role assignment triggers
    7. **07_fallas.sql** - Creates failure tracking table
    8. **08_rls_policies.sql** - Sets up Row Level Security policies
    9. **09_semestres.sql** - Creates semester management functions

    <Note>
      Copy the contents of each SQL file from `database/supabase/` and paste into the SQL Editor, then click **Run**.
    </Note>
  </Step>

  <Step title="Get API Credentials">
    1. In your Supabase project dashboard, click **Settings** → **API**
    2. Copy the following values:
       - **Project URL** (e.g., `https://xxxxx.supabase.co`)
       - **anon public** API key
  </Step>
</Steps>

## Step 3: Configure Environment Variables

<Steps>
  <Step title="Create Environment File">
    Create a `.env` file in the root directory:

    ```bash
    cp .env.example .env
    ```
  </Step>

  <Step title="Add Supabase Credentials">
    Edit `.env` and add your Supabase credentials:

    ```bash .env
    VITE_SUPABASE_URL=https://your-project.supabase.co
    VITE_SUPABASE_ANON_KEY=your-anon-key-here
    ```

    Replace `your-project` and `your-anon-key-here` with the values from Step 2.
  </Step>
</Steps>

## Step 4: Load Test Data

<Steps>
  <Step title="Prepare Sample Data">
    The project includes sample CSV files in `database/`:
    - `test_usuarios.csv`
    - `test_estudiantes.csv`
    - `test_empleados.csv`
    - `test_contratistas.csv`
  </Step>

  <Step title="Load via Admin Panel">
    1. Start the development server (see Step 5)
    2. Navigate to `/admin` in your browser
    3. Go to **Semester Management** → **Load CSV**
    4. Upload each CSV file in order
  </Step>
</Steps>

<Tip>
  Alternatively, you can manually insert test data using the SQL editor in Supabase.
</Tip>

## Step 5: Start Development Server

<Steps>
  <Step title="Run the App">
    Start the Vite development server:

    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>
  </Step>

  <Step title="Open in Browser">
    The application will be available at:

    ```
    http://localhost:5173
    ```

    You should see the login page with the UCC institutional branding (blue and orange colors).
  </Step>
</Steps>

## Step 6: Test the Application

<Steps>
  <Step title="Test User Login">
    Try logging in with a test institutional ID:

    1. Enter an institutional ID (e.g., `80123456` if you loaded test data)
    2. Click **INGRESAR**
    3. You should see the user profile page with their roles and information
  </Step>

  <Step title="Report a Failure">
    On the user profile page:

    1. Scroll to the bottom
    2. Click **Reportar TIC**
    3. Select a failure type:
       - **Olvido** (Forgotten) - Allows up to 4 failures
       - **Pérdida** (Lost) - Immediate block
       - **Robo** (Stolen) - Immediate block
    4. Confirm the report
  </Step>

  <Step title="Test Admin Panel">
    Navigate to the admin panel:

    ```
    http://localhost:5173/admin
    ```

    You should see:
    - Real-time dashboard with statistics
    - User list with filter options
    - Reports and export functionality
    - Semester management tools
  </Step>
</Steps>

## Verify Installation

<AccordionGroup>
  <Accordion title="Database Tables Created">
    In Supabase, navigate to **Table Editor** and verify these tables exist:
    - `usuarios`
    - `roles`
    - `usuario_roles`
    - `info_estudiante`
    - `info_empleado`
    - `info_contratista`
    - `registro_fallas`
    - `semestres`

  <Accordion title="Frontend Loads Successfully">
    Check that:

    - No console errors in browser DevTools
    - Vite HMR (Hot Module Replacement) is working

  <Accordion title="API Connection Works">
    Test the Supabase connection:
    1. Open browser DevTools → Network tab
    2. Enter an institutional ID on login page
    3. Look for requests to `*.supabase.co`
    4. Should see 200 OK responses
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Supabase Connection Error">
    If you see "Failed to fetch" or CORS errors:

    - Verify `.env` file has correct `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`
    - Ensure environment variables start with `VITE_` prefix (required by Vite)
    - Restart the dev server after changing `.env`
    - Check Supabase project status (should be "Active")
  </Accordion>

  <Accordion title="User Not Found">
    If login returns "ID no encontrado":

    - Verify test data was loaded into `usuarios` table
    - Check table has records: Go to Supabase → Table Editor → `usuarios`
    - Try a different institutional ID from your test data
  </Accordion>

  <Accordion title="Port Already in Use">
    If port 5173 is already in use:

    ```bash
    # Vite will automatically use the next available port
    # Or specify a different port:
    npm run dev -- --port 3000
    ```
  </Accordion>

  <Accordion title="Database Migration Failed">
    If SQL execution fails:

    - Execute files in the correct order (01 through 09)
    - Check for syntax errors in the SQL output
    - Ensure you're using the SQL Editor, not the API
    - Try dropping tables and re-running from scratch
  </Accordion>
</AccordionGroup>
