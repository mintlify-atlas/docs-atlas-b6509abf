## Core Tables

### usuarios

Main user registry containing basic identification and access control data.

<ParamField path="id" type="BIGINT" required>
  Primary key. Auto-generated identity column.
</ParamField>

<ParamField path="id_institucional" type="VARCHAR(20)" required>
  Institutional ID assigned by UCC. Must be unique.
  
  **Example:** `80100001`, `80100002`
</ParamField>

<ParamField path="documento_identidad" type="VARCHAR(20)" required>
  National ID number (Cédula de Ciudadanía). Must be unique.
  
  **Example:** `1037654321`
</ParamField>

<ParamField path="nombre_completo" type="VARCHAR(150)" required>
  Full name of the user.
  
  **Example:** `Juan Diego Pérez Gómez`
</ParamField>

<ParamField path="acceso" type="estado_de_acceso" default="activo">
  Access status. ENUM type with values:
  - `activo` - User can enter
  - `bloqueado` - Access denied
</ParamField>

<ParamField path="total_fallas" type="INT" default="0">
  Accumulated failure count. Maximum 4 before automatic block.
</ParamField>

**Indexes:**
- `idx_usuarios_id_institucional` on `id_institucional`
- `idx_usuarios_documento` on `documento_identidad`
- `idx_usuarios_acceso` on `acceso`

**SQL Definition:**
```sql
CREATE TABLE usuarios (
    id                  BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_institucional    VARCHAR(20)       NOT NULL UNIQUE,
    documento_identidad VARCHAR(20)       NOT NULL UNIQUE,
    nombre_completo     VARCHAR(150)      NOT NULL,
    acceso              estado_de_acceso  NOT NULL DEFAULT 'activo',
    total_fallas        INT               NOT NULL DEFAULT 0
);
```

### roles

Fixed catalog of available roles in the system.

<ParamField path="id" type="BIGINT" required>
  Primary key. Auto-generated identity column.
</ParamField>

<ParamField path="nombre_rol" type="VARCHAR(50)" required>
  Role name. Must be unique.
  
  **Values:**
  - `Estudiante`
  - `Empleado`
  - `Contratista`
</ParamField>

<ParamField path="descripcion" type="TEXT">
  Description of the role.
</ParamField>

<Note>
  This table is pre-populated with three roles and should not be modified during regular operations.
</Note>

**Initial Data:**
```sql
INSERT INTO roles (nombre_rol, descripcion) VALUES
    ('Estudiante',   'Usuario matriculado en un programa académico'),
    ('Empleado',     'Trabajador vinculado directamente a la UCC'),
    ('Contratista',  'Proveedor externo con contrato activo');
```

### usuario_roles

Many-to-many relationship table between users and roles.

<ParamField path="id" type="BIGINT" required>
  Primary key. Auto-generated identity column.
</ParamField>

<ParamField path="id_institucional" type="VARCHAR(20)" required>
  Foreign key to `usuarios.id_institucional`.
  
  **Cascade:** `ON DELETE CASCADE`
</ParamField>

<ParamField path="rol_id" type="BIGINT" required>
  Foreign key to `roles.id`.
</ParamField>

**Constraints:**
- `UNIQUE (id_institucional, rol_id)` - Prevents duplicate role assignments

**Indexes:**
- `idx_usuario_roles_institucional` on `id_institucional`
- `idx_usuario_roles_rol` on `rol_id`

<Warning>
  This table is automatically populated by triggers. Do not insert records manually.
</Warning>

**SQL Definition:**
```sql
CREATE TABLE usuario_roles (
    id               BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_institucional VARCHAR(20) NOT NULL REFERENCES usuarios(id_institucional) ON DELETE CASCADE,
    rol_id           BIGINT      NOT NULL REFERENCES roles(id),
    CONSTRAINT uq_usuario_rol UNIQUE (id_institucional, rol_id)
);
```

## Transaction Tables

### fallas

Records incidents when users don't have their institutional ID card (TIC).

<ParamField path="id" type="BIGINT" required>
  Primary key. Auto-generated identity column.
</ParamField>

<ParamField path="id_institucional" type="VARCHAR(20)" required>
  Foreign key to `usuarios.id_institucional`.
  
  **Cascade:** `ON DELETE CASCADE`
</ParamField>

<ParamField path="fecha_hora" type="TIMESTAMPTZ" default="NOW()">
  Timestamp of the incident (with timezone).
</ParamField>

<ParamField path="motivo" type="VARCHAR(10)" required>
  Reason for the failure. CHECK constraint allows only:
  - `olvido` - User forgot the card
  - `perdida` - Card is lost
</ParamField>

**Triggers:**
- `trg_fallas_insert` - Recalculates total_fallas and blocks user if needed
- `trg_fallas_delete` - Recalculates total_fallas if a failure is removed

**Indexes:**
- `idx_fallas_id_inst` on `id_institucional`
- `idx_fallas_fecha_hora` on `fecha_hora DESC`

**SQL Definition:**
```sql
CREATE TABLE fallas (
    id               BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_institucional VARCHAR(20)  NOT NULL
                     REFERENCES usuarios(id_institucional) ON DELETE CASCADE,
    fecha_hora       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    motivo           VARCHAR(10)  NOT NULL
                     CHECK (motivo IN ('olvido', 'perdida'))
);
```

### semestres

Manages academic semester periods.

<ParamField path="id" type="BIGSERIAL" required>
  Primary key. Auto-generated.
</ParamField>

<ParamField path="nombre" type="TEXT" required>
  Semester name.
  
  **Example:** `2026-1`, `2026-2`
</ParamField>

<ParamField path="fecha_inicio" type="DATE" required>
  Semester start date.
</ParamField>

<ParamField path="fecha_fin" type="DATE" required>
  Semester end date.
</ParamField>

<ParamField path="activo" type="BOOLEAN" default="true">
  Whether this semester is currently active.
</ParamField>

<ParamField path="creado_en" type="TIMESTAMPTZ" default="now()">
  Creation timestamp.
</ParamField>

**Constraints:**
- `semestres_fechas_validas` - Ensures `fecha_fin > fecha_inicio`
- Unique index on `activo` where `activo = true` - Only one active semester allowed

**SQL Definition:**
```sql
CREATE TABLE semestres (
  id           BIGSERIAL PRIMARY KEY,
  nombre       TEXT        NOT NULL,
  fecha_inicio DATE        NOT NULL,
  fecha_fin    DATE        NOT NULL,
  activo       BOOLEAN     NOT NULL DEFAULT true,
  creado_en    TIMESTAMPTZ NOT NULL DEFAULT now(),
  CONSTRAINT semestres_fechas_validas CHECK (fecha_fin > fecha_inicio)
);

CREATE UNIQUE INDEX semestres_activo_unico
  ON semestres (activo)
  WHERE activo = true;
```
