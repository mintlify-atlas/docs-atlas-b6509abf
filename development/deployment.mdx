## Overview

This guide covers deploying the UCC Control de Acceso application to production. The application uses **Vite** for building and **Supabase** for backend infrastructure.

<Note>
  Before deploying to production, ensure you have completed thorough testing in a staging environment.
</Note>

## Deployment Architecture

```
┌────────────────────────────────────────┐
│            Users (Browser)              │
└────────────────────────────────────────┘
                    │
                    ↓ HTTPS
┌────────────────────────────────────────┐
│      CDN (Cloudflare/Vercel)          │
└────────────────────────────────────────┘
                    │
                    ↓
┌────────────────────────────────────────┐
│    Static Hosting (Vercel/Netlify)    │
│        Frontend (React + Vite)         │
└────────────────────────────────────────┘
                    │
                    ↓ API Calls
┌────────────────────────────────────────┐
│         Supabase Cloud               │
│  (PostgreSQL + Auth + Storage)      │
└────────────────────────────────────────┘
```

## Building for Production

### Create Production Build

```bash
# Install dependencies
npm install

# Run production build
npm run build
```

This creates an optimized build in the `dist/` directory:

```
dist/
├── assets/
│   ├── index-[hash].js      # Bundled JavaScript
│   ├── index-[hash].css     # Bundled CSS
│   └── vendor-[hash].js     # Vendor libraries
├── index.html               # Entry HTML file
└── favicon.ico              # Favicon
```

### Build Configuration

```javascript vite.config.js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  build: {
    outDir: 'dist',
    sourcemap: false, // Disable in production
    minify: 'terser',
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor': ['react', 'react-dom', 'react-router-dom'],
          'supabase': ['@supabase/supabase-js'],
          'charts': ['recharts'],
          'excel': ['xlsx']
        }
      }
    }
  }
});
```

## Deployment Options

### Option 1: Vercel (Recommended)

<Steps>
  <Step title="Install Vercel CLI">
    ```bash
    npm install -g vercel
    ```
  </Step>

  <Step title="Login to Vercel">
    ```bash
    vercel login
    ```
  </Step>

  <Step title="Deploy">
    ```bash
    vercel --prod
    ```
  </Step>
</Steps>

#### Vercel Configuration

Create `vercel.json`:

```json vercel.json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ],
  "env": {
    "VITE_API_URL": "@api-url",
    "VITE_SUPABASE_URL": "@supabase-url",
    "VITE_SUPABASE_KEY": "@supabase-key"
  }
}
```

<Tip>
  **Vercel Benefits**: Zero-config deployment, automatic HTTPS, global CDN, and instant rollbacks.
</Tip>

### Option 2: Netlify

<Steps>

```toml netlify.toml

  command = "npm run build"

  to = "/index.html"
  status = 200

[build.environment]

```

### Option 4: Docker Container

Create `Dockerfile`:

```dockerfile Dockerfile
# Build stage
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

# Production stage
FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

Nginx configuration:

```nginx nginx.conf
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /assets {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

Build and run:

```bash
# Build image
docker build -t ucc-control-acceso .

# Run container
docker run -p 8080:80 ucc-control-acceso
```

## Environment Variables

### Production Environment

Set these environment variables in your deployment platform:

```bash .env.production
# API Configuration
VITE_API_URL=https://api.ucc-control.com

# Supabase Configuration
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here

# Environment
VITE_ENVIRONMENT=production
```

<Warning>
  **Never commit `.env` files to version control!** Use your deployment platform's environment variable settings.
</Warning>

## Database Setup (Supabase)

### Production Database

<Steps>
  <Step title="Create Supabase Project">
    Go to [supabase.com](https://supabase.com) and create a production project.
  </Step>

  <Step title="Run Migrations">
    Execute SQL migrations in order:
    
    ```bash
    01_usuarios.sql
    02_roles.sql
    03_info_estudiante.sql
    04_info_empleado.sql
    05_info_contratista.sql
    06_triggers_roles.sql
    07_fallas.sql
    08_rls_policies.sql
    09_semestres.sql
    ```
  </Step>

  <Step title="Configure RLS Policies">
    Ensure Row Level Security is enabled and policies are correctly configured.
  </Step>

  <Step title="Create Indexes">
    ```sql
    CREATE INDEX idx_usuarios_id_institucional ON usuarios(id_institucional);
    CREATE INDEX idx_usuarios_acceso ON usuarios(acceso);
    CREATE INDEX idx_fallas_usuario ON registro_fallas(usuario_id);
    CREATE INDEX idx_fallas_fecha ON registro_fallas(fecha_hora DESC);
    ```
  </Step>

  <Step title="Backup Strategy">
    Configure automated backups in Supabase dashboard:
    - Daily automated backups
    - Point-in-time recovery enabled
    - Backup retention: 30 days
  </Step>
</Steps>

## CI/CD Pipeline

### GitHub Actions

Create `.github/workflows/deploy.yml`:

```yaml .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
      
      - name: Build
        run: npm run build
        env:
          VITE_API_URL: ${{ secrets.VITE_API_URL }}
          VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
          VITE_SUPABASE_KEY: ${{ secrets.VITE_SUPABASE_KEY }}
      
      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Deployment fails">
    Check:
    - Build logs for errors
    - Environment variables are set
    - Dependencies are installed
    - Build command is correct
  </Accordion>

  <Accordion title="Site is slow">
    Investigate:
    - Bundle size (use `npm run build -- --analyze`)
    - Database query performance
    - CDN caching
    - Image optimization
  </Accordion>

  <Accordion title="Database connection fails">
    Verify:
    - Supabase URL and key are correct
    - RLS policies allow access
    - Network connectivity
    - Database is not paused (Supabase free tier)
  </Accordion>

  <Accordion title="Routes not working">
    Ensure:
    - Redirects are configured for SPA
    - `vercel.json` or `netlify.toml` is correct
    - All routes fallback to `index.html`
  </Accordion>
</AccordionGroup>
