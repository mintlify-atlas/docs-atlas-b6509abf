## Overview

The UCC Control de Acceso frontend is built with **React 18.2** using a modular, feature-based architecture. The application follows Clean Architecture principles with clear separation of concerns.

## Technology Stack

| Technology | Version | Purpose |
|------------|---------|----------|
| **React** | 18.2.0 | UI framework |
| **React Router** | 6.20.0 | Client-side routing |
| **Vite** | 5.0.8 | Build tool and dev server |
| **Tailwind CSS** | 3.3.6 | Utility-first CSS framework |
| **Axios** | 1.6.2 | HTTP client |
| **Recharts** | 3.7.0 | Data visualization |
| **XLSX** | 0.18.5 | Excel export functionality |
| **Supabase** | 2.97.0 | Database client |

## Routing Architecture

The application uses **React Router v6** for navigation:

```jsx src/App.jsx
import { BrowserRouter as Router, Routes, Route, Navigate } from 'react-router-dom';
import { ProtectedRoute } from './shared/components/ProtectedRoute';

function App() {
  return (
    <Router>
      <Routes>
        {/* Public route */}
        <Route path="/login" element={<LoginPage />} />

        {/* Protected route */}
        <Route
          path="/user-profile"
          element={
            <ProtectedRoute>
              <UserProfilePage />
            </ProtectedRoute>
          }
        />

        {/* Admin route */}
        <Route path="/admin" element={<AdminPage />} />

        {/* Default redirect */}
        <Route path="/" element={<Navigate to="/login" replace />} />
      </Routes>
    </Router>
  );
}
```

### Route Protection

Protected routes use the `ProtectedRoute` component:

```jsx src/shared/components/ProtectedRoute.jsx
export const ProtectedRoute = ({ children }) => {
  const user = localStorage.getItem('current_user');
  
  if (!user) {
    return <Navigate to="/login" replace />;
  }
  
  return children;
};
```

## Data Fetching

The application uses multiple approaches for data fetching:

### 1. Axios HTTP Client

Configured with interceptors for authentication:

```javascript src/modules/login/infrastructure/api/apiClient.js
import axios from 'axios';

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_URL || 'http://localhost:3000',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
});

// Request interceptor - add auth token
apiClient.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('auth_token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// Response interceptor - handle errors
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('auth_token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

### 2. Supabase Client

Direct database access for certain operations:

```javascript src/shared/lib/supabaseClient.js
import { createClient } from '@supabase/supabase-js';

const SUPABASE_URL = 'https://zhljlbuwpezwgkpwvkeu.supabase.co';
const SUPABASE_KEY = process.env.VITE_SUPABASE_KEY;

export const supabase = createClient(SUPABASE_URL, SUPABASE_KEY);
```

### 3. Repository Pattern

Abstracting data access behind repositories:

```javascript
export class AdminRepositoryImpl {
  async getStats() {
    const { data, error } = await supabase
      .rpc('get_dashboard_stats');
    
    if (error) throw error;
    return data;
  }

  async getUsuarios() {
    const { data, error } = await supabase
      .from('usuarios')
      .select('*');
    
    if (error) throw error;
    return data;
  }

  async toggleAcceso(id_institucional, nuevoAcceso) {
    const { error } = await supabase
      .from('usuarios')
      .update({ acceso: nuevoAcceso })
      .eq('id_institucional', id_institucional);
    
    if (error) throw error;
  }
}
```

### Custom Hooks

Encapsulate reusable logic:

```javascript
export const useLogin = () => {
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  const navigate = useNavigate();

  const login = async (idInstitucional) => {
    setLoading(true);
    setError(null);
    
    try {
      const useCase = new ValidateInstitutionalIdUseCase(repository);
      const result = await useCase.execute(idInstitucional);
      
      if (result.success) {
        localStorage.setItem('current_user', JSON.stringify(result.user));
        navigate('/user-profile');
      } else {
        setError(result.error);
      }
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  return { login, loading, error };
};
```

## Error Handling

### Component-Level Error Boundaries

```jsx
class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    console.error('Error caught:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return <ErrorFallback />;
    }
    return this.props.children;
  }
}
```

### API Error Handling

```javascript
try {
  const response = await apiClient.post('/api/endpoint', data);
  return response.data;
} catch (error) {
  if (error.response) {
    // Server responded with error status
    throw new Error(error.response.data.message);
  } else if (error.request) {
    // Request made but no response
    throw new Error('Sin conexión al servidor');
  } else {
    // Error setting up request
    throw new Error('Error en la solicitud');
  }
}
```
