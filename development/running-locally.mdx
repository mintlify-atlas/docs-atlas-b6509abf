## Development Server

The project uses **Vite** as the build tool and development server, providing fast hot module replacement (HMR) and optimized builds.

### Start the Development Server

<CodeGroup>
```bash npm
npm run dev
```

```bash yarn
yarn dev
```

```bash pnpm
pnpm dev
```
</CodeGroup>

The development server will start on `http://localhost:3000` and automatically open in your default browser.

<Note>
  The port can be configured in `vite.config.js` under the `server.port` option.
</Note>

## Available Scripts

The following scripts are defined in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `vite` | Starts the development server with HMR |
| `build` | `vite build` | Creates an optimized production build |
| `preview` | `vite preview` | Previews the production build locally |

### Development Mode

```bash
npm run dev
```

**Features:**
- Hot Module Replacement (HMR) - changes appear instantly
- Source maps enabled for debugging
- Fast refresh for React components
- Development-specific error messages

### Production Build

```bash
npm run build
```

**Output:**
- Optimized JavaScript bundles in `/dist`
- Minified and tree-shaken code
- Source maps generated (configurable)
- Assets are hashed for cache busting

### Preview Production Build

```bash
npm run preview
```

Previews the production build locally before deployment.

## Project Structure

The application follows a **Clean Architecture** pattern with modular organization:

```
src/
├── modules/              # Feature modules
│   ├── login/           # Authentication module
│   │   ├── application/ # Business logic (hooks, use cases)
│   │   ├── domain/      # Entities, repositories
│   │   ├── infrastructure/ # API clients, implementations
│   │   └── presentation/   # UI components, pages
│   ├── user-profile/    # User profile module
│   └── admin/           # Admin dashboard module
├── shared/              # Shared utilities
│   ├── components/      # Reusable components
│   └── lib/             # Utilities (Supabase client, etc.)
├── assets/              # Static assets (images, icons)
├── App.jsx              # Root application component
├── index.jsx            # Application entry point
└── index.css            # Global styles
```

## Application Routes

The application defines the following routes:

| Route | Component | Description | Protected |
|-------|-----------|-------------|----------|
| `/` | Redirect to `/login` | Root redirect | No |
| `/login` | `LoginPage` | User authentication | No |
| `/user-profile` | `UserProfilePage` | User profile and failure registration | Yes |
| `/admin` | `AdminPage` | Admin dashboard | Yes |

<Note>
  Protected routes require authentication. See `src/shared/components/ProtectedRoute.jsx` for implementation.
</Note>

    Open `http://localhost:3000/login` in your browser.

  </Step>

  </Step>

## Hot Module Replacement (HMR)

Vite provides fast HMR out of the box:

- **CSS changes**: Applied instantly without page reload
- **React components**: Fast refresh preserves component state
- **Module changes**: Updates specific modules without full reload

<Tip>
  If HMR stops working, try refreshing the page or restarting the dev server.
</Tip>

## Debugging

### Browser Console

Open browser DevTools (F12) to:
- View console logs
- Inspect network requests
- Debug JavaScript
- Analyze performance

### VS Code Debugging

Create `.vscode/launch.json`:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "chrome",
      "request": "launch",
      "name": "Launch Chrome",
      "url": "http://localhost:3000",
      "webRoot": "${workspaceFolder}/src"
    }
  ]
}
```

### Network Debugging

Monitor API calls:

```javascript
// Check axios interceptors in src/modules/login/infrastructure/api/apiClient.js
apiClient.interceptors.request.use((config) => {
  console.log('Request:', config);
  return config;
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 already in use">
    Change the port in `vite.config.js`:
    ```javascript
    server: {
      port: 3001
    }
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    Clear cache and reinstall:
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="HMR not working">
    1. Check browser console for errors
    2. Restart the dev server
    3. Clear browser cache
    4. Check for syntax errors in code
  </Accordion>

  <Accordion title="Supabase connection errors">
    Verify:
    1. `.env` file has correct Supabase credentials
    2. Supabase project is active
    3. Network connectivity
    4. Check `src/shared/lib/supabaseClient.js`
  </Accordion>
</AccordionGroup>
